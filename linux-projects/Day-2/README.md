# Day 02 - Users and Permissions

## 🎯 Objective

Linux is a multi-user operating system, meaning multiple users can access the same system simultaneously. Proper user management and file permissions are critical for maintaining security, controlling access, and managing resources.

This module introduces Linux user accounts, groups, file ownership, permission management, and administrative privileges.

---

## Topics Covered

- Understanding Linux Users
- User Management Commands
- Linux Groups
- File Ownership
- chmod
- chown
- umask
- sudo
- Practical Lab

---

## Learning Outcomes

After completing this module, you will be able to:

- Create and manage Linux users
- Create and manage groups
- Change file ownership
- Modify file permissions
- Understand default permissions
- Grant administrative privileges safely

---

## Why This Matters in DevOps

DevOps engineers routinely manage Linux servers where multiple users, services, and applications coexist. Proper permission management is essential to:

- Secure production servers
- Control deployment access
- Configure application ownership
- Prevent unauthorized changes
- Manage CI/CD users and service accounts
