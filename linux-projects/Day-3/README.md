# Day 03 - Linux Processes and Services

## 🎯 Objective

Every program running on Linux is executed as a process. Managing processes and system services is a core responsibility of Linux Administrators and DevOps Engineers.

This module covers process management, monitoring system resources, controlling services using systemd, viewing logs, and automating recurring tasks using cron.

---

## Topics Covered

- Linux Processes
- Process Monitoring
- Process Signals
- System Services (systemctl)
- System Logs
- Cron Jobs
- Practical Lab

---

## Learning Outcomes

After completing this module, you will be able to:

- Understand how Linux processes work
- Monitor CPU and memory usage
- Start, stop, and restart services
- Read application and system logs
- Schedule automated tasks using cron

---

## Why This Matters in DevOps

Production servers run hundreds of processes and services. DevOps engineers must monitor application health, restart failed services, investigate logs, and automate routine maintenance tasks.
